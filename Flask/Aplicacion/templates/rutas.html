Rutas

Primer argumento

Las rutas en flask se manejan con el decorador .route(), estas como minimo deben de llevar un argumento con una barra ('/'). Estas nos ayudan a definir lo que va a ir en el navegador, todas las rutas al principio deben de llevar la barra, como ahorita el codigo esta pagina es: 

    
      @app.route('/rutas')
        def rutas():
            ...
            return

Por lo que el navegador tendra el.dominio.com/rutas

Tambien se le pueden asignar varias rutas a la misma funcion, simplemente acumulando decoradores: 

    
      @app.route('/')
      @app.route('/home')
      @app.route('/index')
      def index():
          return ...

Ya sea a que entres a el.dominio.com/, el.dominio.com/home o el.dominio.com/index, todas van a referirse a la misma pagina.

Segundo argumento

El segundo argumento que se le puede especificar al decorador de la ruta es los metodos que puede recibir, por default puede recibir GET pero si queremos que reciba otros y ademas GET, este tambien lo tenemos que especificar: 

    
      @app.route('/bases', methods=['GET','POST','PUT'])

Ahora esa ruta puede aceptar GET, POST y PUT.

Rutas dinamicas

Ademas podemos tener rutas dinamicas, tenemos que especificarle a la ruta cuantos argumentos y que tipo, ademas que la funcion debe de recibir los parametros, de la siguiente manera: 

    
      @app.route('/rutas/<string:nombre>')
        def rutas(nombre):
            ...
            return

De esta manera le estamos pasando nombre como argumento a la funcion, modifica la ruta de arriba por otra cosa que no sea nombre y veras como funciona:

Se pueden pasar mas de dos argumentos y estos por lo general se hace entre /, de la siguiente manera: 

    
      @app.route('/bases/<string:nombre>/<int:dia>')
        def rutas(nombre, dia):
            ...
            return
